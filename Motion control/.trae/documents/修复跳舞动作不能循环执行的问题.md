# 问题分析

通过分析代码，我发现跳舞动作（case 16）不能循环执行的原因是**超时设置过短**：

1. **跳舞动作实现**：在`main.c`的`executeActionNonBlocking`函数中，跳舞动作使用了一个包含11个步骤的状态机，每个步骤之间有1秒的延迟，完成整个舞蹈需要11秒。

2. **超时设置**：在`robot_config.h`中，`ACTION_TIMEOUT_MS`被设置为5000毫秒（5秒）。

3. **超时检查**：在`executeActionNonBlocking`函数中，有一个超时检查机制（第120-124行），如果动作执行时间超过`ACTION_TIMEOUT_MS`，就会调用`handle_error`函数，将机器人设置为停止状态并将舵机设置到中间位置。

4. **问题结果**：当跳舞动作执行到第6个步骤时（约5秒后），就会触发超时，机器人会突然停止动作并将舵机设置到中间位置，表现为"登腿"现象。

# 解决方案

## 方案1：增加超时时间（推荐）

修改`robot_config.h`文件中的`ACTION_TIMEOUT_MS`值，使其大于跳舞动作的总执行时间：

```c
// 将超时时间从5000毫秒增加到12000毫秒
#define ACTION_TIMEOUT_MS    12000    // 动作超时时间
```

这样可以确保整个舞蹈动作（11秒）能够完整执行，而不会被超时机制中断。

## 方案2：为跳舞动作添加特殊处理

在`executeActionNonBlocking`函数中，为跳舞动作（case 16）添加特殊处理，跳过超时检查：

```c
// 超时检查
if (currentActionState != ACTION_IDLE && g_SystemTime - actionStartTime > ACTION_TIMEOUT_MS && robot.action_mode != 16) {
    currentActionState = ACTION_CANCELLED;
    handle_error(ERROR_ACTION_TIMEOUT); // 处理动作超时错误
    return;
}
```

## 方案3：在每个舞蹈步骤后重置超时计时器

在跳舞动作的每个步骤完成后，重置`actionStartTime`：

```c
// 在每个舞蹈步骤完成后添加
if (danceStep == 0 && g_SystemTime - danceStartTime >= 1000) {
    Action_dongzhuo_2(-9, -9, -9, -9, robot.height, robot.height, robot.height, robot.height, 2, 16);
    danceStep = 1;
    danceStartTime = g_SystemTime;
    actionStartTime = g_SystemTime; // 重置超时计时器
}
// 其他步骤类似处理
```

# 推荐方案

**推荐使用方案1**，因为：
1. 实现简单，只需要修改一个配置值
2. 不会影响其他动作的超时检查
3. 逻辑清晰，易于理解和维护

方案2和方案3需要修改更多的代码，可能会引入新的问题，而且会使代码逻辑变得复杂。
